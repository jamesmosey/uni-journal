Week 3: More HTML markup

Published on October 19, 2023

In week 3, we had a more in-depth look into the various types of lists available to us in HTML. So as well as ordered/unordered lists, we now have access to a different type of list called a 'description' list. Alongside the description list we also learned some more informatiton on nested lists and how to format them.

Although it is relatively uncommon to come across a HTML table nowadays, we learned some table markup for the sake of broadening our understanding of HTML in general. Not too long ago these tables were still quite common, due to CSS support accross different browsers being sub-optimal. However in the modern digital age accessibility is key, and if your website sells a product or service, failing to make your website accessible can have some serious repurcussions. There are multiple reasons as to why opting to use a HTML table for layout instead of the CSS layout techniques:

"Layout tables reduce accessibility for visually impaired users: screen readers, used by blind people, interpret the tags that exist in an HTML page and read out the contents to the user" (MDN Contributers, 2023).

Another reason is that table layouts require more complex markup structures than its CSS counterparts, this can often lead to issues with writing, maintaining and debugging the code.
